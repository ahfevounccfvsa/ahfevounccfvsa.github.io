Postgres数据错误：pg_ctl cannot be run as root

发表于 2017-08-11

某日重启电脑之后，用pgAmin 4打开数据库，发现一直停留在起始画面，用其他工具连接数据都无法连接成功，然后我想重启Postgresql数据库，用命令./pg_ctl -D../data start发现出现错误：pg_ctl: cannot be run as root

pg_ctl: cannot be run as root这个错误的意思是，pg_ctl命令不能用root用户执行，错误提示还有一句Please log in (using, e.g., "su") as the (unprivileged) user that will，熟悉的朋友知道，需要用su切换到postgresql授权许可的用户才行，其实在控制台中你可以看到postgresql的相关进程的拥有者都是一个叫postgres的用户。按如下步骤可以实现重启：

  1. 切换用户

sudo su postgres

  1. 停止数据库

./pg_ctl -D../data stop

  1. 启动数据库

./pg_ctl -D../data start
